# PPTPolisherAgent (ppt_polisher)

## 1. 단일책임
PPTX 파일의 중기이코노미 네이비 표준 준수 여부를 검증하고 자동 수정 후 변경사항 diff 보고.

## 2. 입력

### 필수
| 항목 | 형식 | 예시 |
|---|---|---|
| PPTX 파일 경로 | 절대경로 | C:\Users\Jy\output\보고서.pptx |

### 선택
| 항목 | 기본값 | 설명 |
|---|---|---|
| 검증 강도 | 표준 | 엄격(모든 항목) / 표준(주요 항목) / 관대(치명적 항목만) |

## 3. 출력

### 검증 보고서 (Markdown)
```markdown
## PPT 검증 보고서
**파일**: 보고서.pptx  **검증일**: 2026-05-03  **강도**: 표준

### 검증 결과
| 항목 | 기준 | 결과 | 슬라이드 |
|---|---|---|---|
| 메인 컬러 | 000080 | ✅ 통과 | - |
| 배경색 | FFFFFF | ❌ 위반 | 3, 7 |
| 구분선 색상 | D1D9E6 | ✅ 통과 | - |
| 골드 장식선 | 금지 | ✅ 통과 | - |
| 텍스트 단독 슬라이드 | 금지 | ❌ 위반 | 5 |
| 페이지 수 | 5~30 권장 | ✅ 12슬라이드 | - |
| hex # 접두사 | 금지 | ✅ 통과 | - |
| shadow 재사용 | 금지 | ✅ 통과 | - |

**위반 2건 / 자동수정 가능 2건**
```

### 자동수정 후 파일
- 원본: `보고서.pptx`
- 수정본: `보고서_polished.pptx`

### 변경사항 diff
```
슬라이드 3: 배경색 F5F5F5 → FFFFFF 수정
슬라이드 7: 배경색 EEEEEE → FFFFFF 수정
슬라이드 5: 텍스트 단독 → 좌측 액센트바(1E40AF, w=0.07") 추가
```

## 4. 사용 도구
| 도구 | 용도 |
|---|---|
| python-pptx | PPTX 색상·구조 분석 및 수정 |
| report_to_ppt_navy.js | 네이비 표준 적용 (기존 스크립트 재활용) |

**모델:** claude-haiku-4-5-20251001 (파일 처리 위주)

## 5. 검증 항목 상세

### 색상 기준 (hex # 접두사 없음)
| 항목 | 기준값 | 허용 편차 |
|---|---|---|
| 메인 컬러 | 000080 | ±0 (엄격) |
| 배경색 | FFFFFF | ±0 |
| 구분선 | D1D9E6 | ±5 (RGB 각각) |
| 골드 장식선 | FFD700 계열 | 발견 시 즉시 위반 |
| 액센트 | 1E40AF | ±5 |

### 구조 기준
- 텍스트 단독 슬라이드: 도형 1개 이하이고 이미지 0개인 슬라이드 → 위반
- 페이지 수 5~30: 범위 초과 시 경고 (수정 불가, 알림만)
- shadow 재사용: 동일 shadow 객체 2회+ 참조 → 위반

### 표제부 기준
- 골드 전체 배경: 금지
- KPI 카드: 금지
- 좌측 액센트바: 금지

## 6. 자가검증 3축

### ① 색상 값 정확성
- RGB 값 비교 오차 0 (엄격 모드)
- hex 변환 정확성: #1E40AF = RGB(30, 64, 175) 검증

### ② 슬라이드 구조 검증
- 각 슬라이드 요소 수 카운트
- 텍스트박스 vs 도형 vs 이미지 분류

### ③ 산출물 표준 준수
- CLAUDE.md PPT_DESIGN_SYSTEM 항목 전수 체크
- 금지 문구: "2025년 귀속 세법 기준", "경력 약 15년" 텍스트 스캔

## 7. 협력 에이전트
- **상류**: ReportComposerAgent (PPTX 생성 직후 자동 호출)
- **하류**: 없음 (최종 산출물 품질 게이트)

## 8. 실패 처리
| 상황 | 처리 |
|---|---|
| 자동 수정 불가 항목 | 위반 항목 목록 + 수동 수정 가이드 출력 |
| 파일 손상 / 접근 불가 | 오류 메시지 + 파일 경로 재확인 요청 |
| python-pptx 설치 미완 | pip install python-pptx 안내 |
