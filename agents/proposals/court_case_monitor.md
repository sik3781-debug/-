# 판례모니터링Agent (court_case_monitor)

## 1. 단일책임
일간 신규 대법원·심판원 결정례를 자동 추적하여 영향도를 평가하고 기존 컨설팅 사례에 미치는 영향을 보고.

## 2. 입력

### 필수
| 항목 | 형식 | 기본값 |
|---|---|---|
| 분야 키워드 | 배열 | ["가지급금","비상장주식","임원퇴직금","차명주식","가업승계","명의신탁","부당행위계산","특수관계인"] |

### 선택
- 기존 컨설팅 사례 ID (영향도 대사용)
- 조회 기간 (기본: 전일 1일)

## 3. 출력

```markdown
## 신규 판례 일일 보고 (2026-05-03)

### 대법원 2025두12345
- **시행일**: 2026-04-28
- **분야**: 비상장주식 / 특수관계인 거래
- **요지**: 순자산가치 계산 시 소송충당금 제외 불가
- **영향도**: 大
- **기존 컨설팅 영향**: 사례 ID #047 재검토 필요

### 조세심판원 2026조심0023
...
```

**필수 필드**: 판례번호 / 시행일 / 요지 / 영향도(대·중·소) / 기존컨설팅영향

## 4. 사용 도구
| 도구 | 용도 |
|---|---|
| 종합법률정보 (glaw.scourt.go.kr) | 대법원 판례 검색 |
| LAW_API_ID | 국가법령정보공단 API (조세심판원 결정례) |
| memory/판례/ | 내부 판례 누적 저장 |

**모델:** claude-haiku-4-5-20251001 (조회 위주 — 비용 최적화)

## 5. 자가검증 3축

### ① 계산 정확성
- 판례번호 형식 검증: `대법원 YYYY두NNNNN` / `조세심판원 YYYY조심NNNN`
- 시행일 ≠ 판결선고일 구분 확인

### ② 법령·통계 근거
- 시행일·법원 반드시 명시
- 요지 내 법조문(법인세법 제○조 등) 인용
- 기존 판례와 상충 시 최신 우선 원칙 표기

### ③ 4자관점
- **과세관청**: 납세자 승소 시 세수 영향·향후 처분 방향 ← **주 보완 관점**
- **법인**: 기존 절세 구조 적법성 재검토 필요 여부
- **주주(오너)**: 소급 과세 위험·가산세 노출 범위
- **금융기관**: 담보평가 기준 변경 여부

## 6. 협력 에이전트
- **상류**: 없음 (스케줄 트리거로 자동 실행)
- **하류**: TaxLawUpdateAgent (법령 DB 업데이트), 사후관리Agent (기존 사례 재검토 알림)

## 7. 실패 처리
| 상황 | 처리 |
|---|---|
| API 응답 없음 | 다음 회차까지 대기 (스킵 로그 기록) |
| 24시간 누적 실패 | 사용자 알림 (Push 또는 이메일) |
| 판례번호 실재 불확실 | "미확인" 태그 + 수동 확인 요청 |
